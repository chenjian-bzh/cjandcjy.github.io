tomcat启动超时错误

By xiao jian

2016-06-29 Updated:2016-06-29

Contents

The APR based Apache Tomcat Native library which allows optimal performance in production environments was not found

tomcat启动时的警告，导致一个项目总是启动超时，试着在eclipse里的server设置里面修改了tomcat的启动超时时间，，默认是45秒，我把它改成了120秒，结果还是超时。。。然后换另一个项目放到tomcat中启动，又报了其他的错误，具体错误信息忘记记下来了，只好重装tomcat，重装好了之后其他项目可以运行，但是bus项目依然启动超时，控制台信息显示是

信息: Initializing Spring FrameworkServlet 'springmvc'

就一直卡在这不动了，显然是在初始化spring容器时除了问题，就试着update项目，设置项目的字符集为utf-8，在部署真的好了，至于造成错误的具体原因还不太清楚。
再运行的时候就可以了。
