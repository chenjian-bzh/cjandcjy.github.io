泛型的意义

关于泛型出现的意义我们看一看以下的例子：

class Automobile{}

public class Holder{
   private Automobile automobile;

   public void get(){
      return automobile;
   }

   public void set(Automoblie auto){
     this.automobile = auto;
   }
}

上述代码中Holder容器类只能存储一种对象，那就是Automobile类的实例，无法持有其他任何类型的对象，而我们并不希望为每一个类编写一个容器类，下面我们对Holder进行改写

public class Holder{
       private Object object;

       public void get(){
          return object;
       }

       public void set(Object object){
         this.object = object;
       }
}

这样的Holder类就可以存储任何类型的对象了，在有些情况下，我们确实希望容器类可以存储多种不同类型的对象，但是通常而言，我们只会让一个容器存储一种类型的变量，这就是泛型出现的意义，它可以指定容器存储的对象的类型，而且由编译器来保证类型的正确性。与其用Object宽泛的指定存储类型，不如在声明容器类的时候不指定类型，稍后再决定使用什么类型。泛型类的写法是这样的，使用尖括号包裹类型参数，放在类名之后，在使用的时候用指定的类型代替类型参数即可，如下所示：

public class Holder<T>{
           private T t;

           public void get(){
              return t;
           }

           public void set(T t){
             this.t = t;
           }
}

这样在你创建Holder对象的时候必须指定类型，之后只有该类型的对象可以存入，且取出的时候自动就是该类型的对象了，无需进行强制类型转换。

泛型方法

泛型类在使用过程中必须传入类型参数，而使用泛型方法的时候不需要指定参数类型，因为编译器会为我们找出具体的类型，这称为类型参数推断。所以我们可以像调用不同方法一样调用泛型方法，就好像它被无数次重载过一样；泛型方法使得方法独立于类而产生变化，且有一个原则，无论何时，当可以使用泛型方法的时候，就应当避免整个类的泛型化，另外由于static方法无法访问类型参数，所以如果想让static方法具有泛型能力，只能使其成为泛型方法。

public static  <T,K,V> void get(T t,K k,V v){

   System.out.println(t+","+k+","+v);       
}

public static void main(String[] args) {

    EhCache.get("generic", 1, 2.0f);  //直接传入参数，无需指定类型
}

泛型方法和可变参数列表可以很好的用在一起：

public static <T> List<T> getlist(T... args){
    List<T> list = new ArrayList<>();
    for(T t:args){
        list.add(t);
    }
    return list;
}

public static void main(String[] args) {

    List<String> listA = EhCache.getlist("A");

    List<String> listB = EhCache.getlist("A","B","C");

    List<String> listC = EhCache.getlist("A,B,C,D,E,F,G,H,I".split(","));
}

注意

java泛型的一个局限性是无法使用基本数据类型作为类型参数，不过自java se5之后有了自动装箱和拆箱，可以方便的在基本数据类型和她们的包装类之间转换
